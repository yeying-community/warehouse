# S3 兼容接口实施清单（可执行）

本文将 [S3兼容接口设计.md](./S3兼容接口设计.md) 拆解为可执行任务。

目标是在不替换当前本地文件存储的前提下，为 Warehouse 增加可通过 AWS CLI、AWS SDK for Go 和 rclone 使用的 S3 兼容接口，并确保权限、配额、原子写入和 active/standby 复制规则不被绕过。

## 使用说明

- 本清单描述推荐实施顺序，不代表对应功能已经完成。
- 每个里程碑必须独立可验证，不能把所有改造积累到最后一次性联调。
- 阶段 1 是内部小文件 MVP，不作为普通用户生产开放版本。
- 完成 Multipart、并发配额、中断清理和目标客户端测试后，才满足生产开放条件。
- S3 相关功能默认关闭，通过 `s3.enabled` 灰度启用。

## 0. 已冻结决策

- [x] S3 是协议适配层，不替换本地文件系统。
- [x] 使用独立域名 `https://s3.yeying.pub`。
- [x] 使用独立内部监听 `127.0.0.1:6066`。
- [x] region 使用 `us-east-1`。
- [x] 第一阶段只支持 path-style。
- [x] 第一阶段只开放 `personal` 和 `apps` bucket。
- [x] 分享 bucket 延后。
- [x] WebDAV Basic 密钥和 S3 Signature V4 密钥严格分开。
- [x] 第一阶段不实现 Presigned URL。
- [x] 第一批兼容客户端为 AWS CLI、AWS SDK for Go、rclone。
- [x] S3 DeleteObject 永久删除。
- [x] WebDAV 和 Warehouse 页面删除继续进入回收站。
- [x] 第一阶段不保存任意 `x-amz-meta-*`，仅特殊支持 `x-amz-meta-mtime`。
- [x] 第一阶段 S3 流量只进入 active。

## 1. 里程碑与完成定义

### 1.1 里程碑

1. M0：协议验证和代码边界冻结。
2. M1：统一对象操作基础能力完成。
3. M2：S3 凭证和 Signature V4 完成。
4. M3：小文件 MVP 完成。
5. M4：Multipart 和生产大文件能力完成。
6. M5：代理、可观测性、灰度和运维能力完成。
7. M6：分享 bucket 和增强能力，后续单独评审。

### 1.2 生产开放 DoD

- AWS CLI、AWS SDK for Go、rclone 兼容测试通过。
- S3 和 WebDAV 可以交叉读取彼此写入的文件。
- PutObject、覆盖、DeleteObject、CopyObject、Multipart 均经过权限校验。
- 并发上传不能绕过用户额度。
- 上传中断不会暴露不完整正式对象。
- 未完成 Multipart 能够自动清理。
- S3 正式写入能够触发 `MutationRecorder`。
- standby 不参与外部写入和 quota 拒绝。
- active 切换后，已完成并复制的对象可继续读取。
- 日志不包含 Access Key Secret、Authorization 和签名。
- S3 关闭后不影响现有 WebDAV、API 和前端。

## 9. 推荐开发顺序

1. 先做 M0 的签名和代理技术验证。
2. 再做 M1 的 ObjectService、路径、权限和 quota reserve。
3. 然后做 M2 的凭证和 Signature V4。
4. 完成 M3 小文件闭环，在测试环境验证交叉协议。
5. 完成 M4 Multipart 后再进入生产灰度。
6. 最后补 M5 运维能力并逐步开放。

不建议先写 S3 Handler 再补 ObjectService，否则容易形成绕过 quota、回收站和复制机制的第二套写入链路。

## 10. 风险清单

- 风险：代理修改 URI 导致签名失败。应对：独立域名，不重写 URI/query，增加 Nginx 后端到端签名测试。

- 风险：S3 Secret 必须可解密，数据库泄漏风险高于 bcrypt。应对：AES-256-GCM、外部主密钥、key version、禁止日志输出。

- 风险：并发上传绕过 quota。应对：数据库原子 reserve + key 提交锁 + quota reconcile。

- 风险：Multipart staging 占满磁盘。应对：临时占用计费、数量限制、大小限制、24h 自动清理。

- 风险：WebDAV 与 S3 对同一个 key 并发写入。应对：统一 ObjectService 后使用同一 key 锁；完成统一前限制 S3 灰度范围。

- 风险：S3 永久删除造成误操作。应对：S3 密钥创建提示、最小权限绑定、删除权限默认不勾选。

- 风险：standby 读取到旧数据。应对：第一阶段 S3 流量只进入 active。
