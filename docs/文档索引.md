# WebDAV 设计文档索引

本目录包含基于当前代码实现整理的设计文档，面向开发/维护人员。

快速开始：`README.md`

## 推荐阅读顺序

如果你第一次接手这个项目，建议按下面顺序阅读：

1. `README.md`：本地启动、目录结构、最短验证路径
2. `架构概览.md`：先建立整体模块视角
3. `认证设计.md`：理解 Basic / JWT / UCAN / 访问密钥
4. `WebDAV与相关接口API（简明版）.md`：查看外部调用方式
5. `部署手册.md`：查看部署、Nginx、active/standby 上线方式
6. `容灾方案.md`：理解当前高可用现状、术语与运维边界

## standby / 容灾阅读路径

如果你当前只关心 active / standby、高可用部署、复制排障，建议按下面顺序阅读：

1. `部署手册.md`
   - 先看怎么启动、怎么配置、怎么观察、怎么排障
   - 这是运维入口
2. `容灾方案.md`
   - 再看当前选定路线、术语、核心表职责、发现时序、fan-out 时序
   - 这是“当前实现到底怎么工作”的说明
3. `内部复制版备用节点设计.md`
   - 最后看内部复制链路细节、表结构、状态机、generation/fence/reconcile/bootstrap 设计
   - 这是研发和深度排障入口
4. `多实例与多副本方案设计.md`
   - 如果要继续往自动 failover / rebalance / 更复杂多副本编排演进，再看这份
   - 这是未来路线和方案比较，不是当前运行手册

一句话区分：

- `部署手册.md` 解决“怎么用、怎么查、怎么恢复”
- `容灾方案.md` 解决“现在实际上怎么跑、边界是什么”
- `内部复制版备用节点设计.md` 解决“代码和数据结构为什么这么设计”
- `多实例与多副本方案设计.md` 解决“以后还可以怎么演进”
