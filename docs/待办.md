## P1 阶段二剩余增强项

### 1. 旧 generation 数据清理与归档

目标：

- 既然 generation 已经严格生效，就要补上历史代际数据的生命周期管理

待做项：

- [ ] 为 `replication_outbox` 增加旧 generation 清理策略
- [ ] 为 `replication_reconcile_jobs` / `replication_reconcile_items` 增加保留期与清理策略
- [ ] 明确 `replication_offsets` 在 assignment 释放或切换后的保留策略
- [ ] 明确 `cluster_replication_assignments` 的 released / expired 记录是否归档或裁剪
- [ ] 设计人工触发与自动清理两种方式

验收标准：

- 长期运行后数据库不会因为历史 generation 累积而无限膨胀

### 2. reconcile 调度与自愈

目标：

- 在已具备“startup reconcile + periodic auto reconcile sweep”的基础上
- 继续补齐更完整的历史补齐自愈能力

待做项：

- [ ] 启动后未完成 reconcile job 的恢复执行
- [ ] reconcile 失败后的更细粒度节流、分级重试与人工暂停策略
- [ ] reconcile 差异比较与跳过逻辑，减少无效传输

验收标准：

- standby 中断、重启、长时间落后后，重新上线能够自动恢复到一致状态

### 3. 观测与告警

目标：

- 当前功能已经能跑，但还缺少“运维可持续观察”的能力

待做项：

- [ ] 复制 lag、failed events、reconcile pending items、assignment lease 状态指标
- [ ] generation mismatch、sequence conflict、assignment 失效等关键错误的结构化日志
- [ ] active / standby 心跳过期告警
- [ ] reconcile 失败与长期未完成告警
- [ ] `ha status` 输出进一步补充对 generation / assignment 状态的解释字段

验收标准：

- 不看数据库明细，也能通过日志、状态接口、指标快速定位复制链路问题
