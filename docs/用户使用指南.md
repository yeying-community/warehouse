# Warehouse 用户使用指南

本文面向 Warehouse 普通用户、集成方和运维支持人员，说明如何使用 Web 页面、WebDAV 凭证和 S3 凭证完成文件管理、客户端接入和验证。

如果你只关心部署、Nginx 或 active/standby，请看 [部署手册.md](./部署手册.md)。如果你要看接口字段和内部设计，请看 [WebDAV与相关接口API（简明版）.md](./WebDAV与相关接口API（简明版）.md) 与 [S3兼容接口设计.md](./S3兼容接口设计.md)。

## 1. 基本概念

### 1.1 资产空间

Warehouse 当前把用户文件分成三个逻辑资产空间：

| 空间 | 用途 | WebDAV 路径 | S3 bucket |
| --- | --- | --- | --- |
| 个人存储 | 用户主动上传、整理和分享的文件 | `/dav/personal/...` | `personal` |
| 应用存储 | DApp、前端应用或 UCAN 场景产生的数据 | `/dav/apps/...` | `apps` |
| 服务存储 | 后台服务集成、定时任务或服务端自动化产生的数据 | `/dav/services/...` | `services` |

这三个空间不是三套存储。WebDAV 和 S3 只是两种访问协议，最终看到的是同一套用户文件。

### 1.2 WebDAV 凭证与 S3 凭证

| 凭证类型 | 用途 | 认证方式 | 是否可互用 |
| --- | --- | --- | --- |
| WebDAV 凭证 | Finder、Windows 网络位置、rclone WebDAV、davfs2、脚本上传 | Basic Auth，用户名是 `keyId`，密码是 `keySecret` | 不能用于 S3 |
| S3 凭证 | AWS CLI、rclone S3、MinIO Client、SDK | AWS Signature V4，Access Key ID + Secret | 不能用于 WebDAV |

所有密钥的 Secret 都只在创建时显示一次。关闭创建弹窗后无法找回，只能撤销旧凭证并重新创建。

### 1.3 权限含义

| 权限 | WebDAV 行为 | S3 行为 |
| --- | --- | --- |
| 读 / 下载 | 列目录、下载文件 | `ListBucket`、`HeadObject`、`GetObject` |
| 新建 / 上传 | 新建目录、上传新文件 | 上传新对象、创建 Multipart 上传 |
| 更新 / 重命名 | 覆盖、移动、重命名 | 覆盖已有对象 |
| 删除 | 删除文件或目录 | 删除对象 |

建议默认使用最小权限：只读任务只给读权限；备份任务通常给读、新建、更新；确实需要清理远端时再给删除权限。

## 2. Web 页面使用

### 2.1 登录与空间切换

1. 打开 Warehouse Web 地址，例如：`https://warehouse.tidukongjian.com/`。
2. 使用账号密码、钱包或站点启用的其他方式登录。
3. 在文件页面切换个人存储、应用存储或服务存储。
4. 进入目录后可以上传、下载、重命名、删除、创建目录和查看详情。

### 2.2 分享

Warehouse 支持两类分享：

- 公开分享：生成可访问链接，适合只读下载。
- 定向分享：分享给指定地址、分组或全员，可按权限允许下载、上传、重命名或删除。

分组分享按当前分组成员动态授权。分组成员变化后，不需要重新创建分享。

### 2.3 回收站

通过 Web 页面或 WebDAV 删除的文件会进入回收站。文件进入回收站后仍占用额度，只有永久删除或清空回收站后才释放额度。

注意：S3 删除遵循 S3 客户端预期，是永久删除，不进入 Warehouse 回收站。

### 2.4 密钥管理

进入「密钥管理」后可以看到两个 tab：

- `WebDAV 凭证`：用于 WebDAV 客户端。
- `S3 凭证`：用于 S3 兼容客户端。

生效中的凭证可以撤销。已撤销的凭证可以删除。删除只用于清理列表，不会恢复 Secret，也不会影响已经失效的访问能力。

## 3. WebDAV 凭证使用

### 3.1 什么时候选 WebDAV

适合这些场景：

- 用系统文件管理器挂载网络盘。
- 用 rclone、davfs2 或 curl 做文件同步。
- 希望删除进入 Warehouse 回收站。
- 需要访问分享、目录树和常规文件管理语义。

### 3.2 创建和绑定 WebDAV 凭证

1. 打开「密钥管理」。
2. 切到 `WebDAV 凭证`。
3. 点击「新建」。
4. 填写密钥名称、权限和有效期。
5. 创建成功后立即保存 `Key ID` 和 `Key Secret`。
6. 到目标目录打开「授权密钥」，把该密钥绑定到具体目录。

如果密钥没有绑定任何目录，WebDAV 认证会失败。建议把绑定目录收窄到具体业务路径，例如 `/personal/backups`，不要默认给整个 `/personal`。

### 3.3 连接信息

请向站点管理员确认以下地址：

```bash
export WAREHOUSE_WEBDAV_URL="https://warehouse.tidukongjian.com/dav/"
export WEBDAV_KEY_ID="ak_xxxxxxxxxxxxxxxx"
export WEBDAV_KEY_SECRET="sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

WebDAV URL 一般是站点域名后加 `/dav/`。如果站点配置了其他 `webdav.prefix`，以管理员提供的地址为准。

### 3.4 curl 验证

列出个人存储根目录：

```bash
curl -i \
  -u "$WEBDAV_KEY_ID:$WEBDAV_KEY_SECRET" \
  -X PROPFIND \
  -H "Depth: 1" \
  "${WAREHOUSE_WEBDAV_URL}personal/"
```

预期结果：HTTP 状态为 `207 Multi-Status`，响应中能看到目录项。

上传并下载一个测试文件：

```bash
printf 'hello warehouse\n' > /tmp/warehouse-webdav-test.txt

curl -i \
  -u "$WEBDAV_KEY_ID:$WEBDAV_KEY_SECRET" \
  -T /tmp/warehouse-webdav-test.txt \
  "${WAREHOUSE_WEBDAV_URL}personal/warehouse-webdav-test.txt"

curl \
  -u "$WEBDAV_KEY_ID:$WEBDAV_KEY_SECRET" \
  "${WAREHOUSE_WEBDAV_URL}personal/warehouse-webdav-test.txt"
```

预期结果：上传返回 `201 Created` 或 `204 No Content`，下载输出 `hello warehouse`。

### 3.5 rclone WebDAV 配置

推荐使用交互式配置，避免把明文 Secret 写入配置文件：

```bash
rclone config
```

按提示选择：

```text
n) New remote
name> warehouse-webdav
Storage> webdav
url> https://warehouse.tidukongjian.com/dav/
vendor> other
user> ak_xxxxxxxxxxxxxxxx
password> sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

验证：

```bash
rclone lsd warehouse-webdav:
rclone lsf warehouse-webdav:personal
rclone copy /tmp/warehouse-webdav-test.txt warehouse-webdav:personal/
rclone cat warehouse-webdav:personal/warehouse-webdav-test.txt
```

### 3.6 davfs2 挂载

Linux 可以使用 davfs2 挂载为本地目录：

```bash
mkdir -p ~/warehouse
sudo mount -t davfs "$WAREHOUSE_WEBDAV_URL" ~/warehouse
```

建议把凭证写入 `~/.davfs2/secrets` 或系统级 secrets 文件，避免命令行泄露：

```text
https://warehouse.tidukongjian.com/dav/ ak_xxxxxxxxxxxxxxxx sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

如果挂载后要像本地目录一样拖拽、覆盖或重命名文件，建议 WebDAV 凭证至少具备读、新建、更新权限。只允许新增且不覆盖的自动化任务，可以只给读、新建。

## 4. S3 凭证使用

### 4.1 什么时候选 S3

适合这些场景：

- 使用 AWS CLI、rclone S3、MinIO Client 或 S3 SDK。
- 做对象式上传、下载、同步和备份。
- 希望工具按 S3 语义处理大文件或对象列表。

当前 S3 是协议兼容层，不是把底层存储迁移到了 AWS S3 或 MinIO。S3 客户端看到的 bucket 是 Warehouse 的逻辑资产空间。

### 4.2 创建 S3 凭证

1. 打开「密钥管理」。
2. 切到 `S3 凭证`。
3. 点击「新建」。
4. 选择逻辑 bucket：`personal`、`apps` 或 `services`。
5. 填写 bucket 下目录，留空表示授权整个 bucket。
6. 创建成功后立即保存 `Access Key ID` 和 `Secret`。

示例：

| 选择 | bucket 下目录 | 实际授权范围 | S3 客户端写法 |
| --- | --- | --- | --- |
| `personal` | 留空 | `/personal` | `s3://personal/...` |
| `personal` | `backup` | `/personal/backup` | `s3://personal/backup/...` |
| `apps` | `demo/data` | `/apps/demo/data` | `s3://apps/demo/data/...` |
| `services` | `reporting/daily` | `/services/reporting/daily` | `s3://services/reporting/daily/...` |

### 4.3 连接信息

请向站点管理员确认 S3 Endpoint 和 Region：

```bash
export WAREHOUSE_S3_ENDPOINT="https://s3.example.com"
export WAREHOUSE_S3_REGION="us-east-1"
export S3_ACCESS_KEY_ID="AKxxxxxxxxxxxxxxxx"
export S3_SECRET_ACCESS_KEY="xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

Warehouse S3 当前推荐 path-style 访问，也就是对象地址形如 `https://s3.example.com/personal/path/to/file.txt`。

### 4.4 AWS CLI 配置与验证

配置环境变量：

```bash
export AWS_ACCESS_KEY_ID="$S3_ACCESS_KEY_ID"
export AWS_SECRET_ACCESS_KEY="$S3_SECRET_ACCESS_KEY"
export AWS_DEFAULT_REGION="$WAREHOUSE_S3_REGION"
```

建议使用独立 profile，并开启 path-style：

```bash
aws configure set profile.warehouse.region "$WAREHOUSE_S3_REGION"
aws configure set profile.warehouse.s3.addressing_style path
```

列 bucket 和对象：

```bash
aws --profile warehouse --endpoint-url "$WAREHOUSE_S3_ENDPOINT" s3 ls
aws --profile warehouse --endpoint-url "$WAREHOUSE_S3_ENDPOINT" s3 ls s3://personal/
```

上传、下载、校验：

```bash
printf 'hello warehouse s3\n' > /tmp/warehouse-s3-test.txt

aws --profile warehouse --endpoint-url "$WAREHOUSE_S3_ENDPOINT" \
  s3 cp /tmp/warehouse-s3-test.txt s3://personal/warehouse-s3-test.txt

aws --profile warehouse --endpoint-url "$WAREHOUSE_S3_ENDPOINT" \
  s3 cp s3://personal/warehouse-s3-test.txt /tmp/warehouse-s3-downloaded.txt

diff /tmp/warehouse-s3-test.txt /tmp/warehouse-s3-downloaded.txt
```

预期结果：`diff` 无输出，表示上传和下载内容一致。

删除测试对象：

```bash
aws --profile warehouse --endpoint-url "$WAREHOUSE_S3_ENDPOINT" \
  s3 rm s3://personal/warehouse-s3-test.txt
```

注意：S3 删除是永久删除，不进入 Warehouse 回收站。

### 4.5 rclone S3 配置与验证

可以通过 `rclone config` 交互式创建，也可以写入配置文件。示例配置：

```ini
[warehouse-s3]
type = s3
provider = Other
env_auth = false
access_key_id = AKxxxxxxxxxxxxxxxx
secret_access_key = xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
endpoint = https://s3.example.com
region = us-east-1
force_path_style = true
```

验证：

```bash
rclone lsd warehouse-s3:
rclone lsf warehouse-s3:personal
rclone copy /tmp/warehouse-s3-test.txt warehouse-s3:personal/
rclone cat warehouse-s3:personal/warehouse-s3-test.txt
```

如果你的 S3 凭证只授权 `/personal/backup`，命令应写到该前缀下：

```bash
rclone copy /tmp/warehouse-s3-test.txt warehouse-s3:personal/backup/
```

### 4.6 MinIO Client 配置与验证

MinIO Client 的命令名通常是 `mc`：

```bash
mc alias set warehouse \
  "$WAREHOUSE_S3_ENDPOINT" \
  "$S3_ACCESS_KEY_ID" \
  "$S3_SECRET_ACCESS_KEY" \
  --api S3v4 \
  --path on
```

验证：

```bash
mc ls warehouse/personal
mc cp /tmp/warehouse-s3-test.txt warehouse/personal/warehouse-s3-test.txt
mc cat warehouse/personal/warehouse-s3-test.txt
mc rm warehouse/personal/warehouse-s3-test.txt
```

### 4.7 S3 能力边界

当前 S3 兼容层适合常见上传、下载、列举、覆盖和删除。需要注意：

- bucket 当前固定为 `personal`、`apps` 和 `services`，不能任意创建新 bucket。
- S3 凭证只能访问创建时选择的 bucket 和目录前缀。
- S3 凭证不访问“分享给我的”目录。
- S3 删除是永久删除，不进入回收站。
- S3 客户端和 WebDAV 客户端可以看到同一份 `personal/apps/services` 数据，但两类凭证不能互用。

## 5. 客户端选择建议

| 场景 | 推荐协议 | 推荐客户端 |
| --- | --- | --- |
| 日常网页上传下载 | Web 页面 | 浏览器 |
| 桌面文件管理器挂载 | WebDAV | macOS Finder、Windows 网络位置、davfs2、支持 WebDAV 的 GUI 客户端 |
| 自动化同步和备份 | WebDAV 或 S3 | rclone |
| S3 工具链或对象式备份 | S3 | AWS CLI、MinIO Client、rclone S3、AWS SDK |
| 希望删除后可从回收站恢复 | WebDAV 或 Web 页面 | 浏览器、WebDAV 客户端 |
| 希望按 S3 客户端语义永久删除 | S3 | AWS CLI、mc、rclone S3 |

选择原则：

- 普通用户优先用 Web 页面。
- 桌面挂载优先用 WebDAV。
- 自动化任务优先用独立凭证，不使用主账号密码。
- 已经使用 S3 工具链的备份、同步或 SDK 集成，使用 S3 凭证。

## 6. 常见问题排查

### 6.1 WebDAV 返回 401

检查：

- `keyId/keySecret` 是否填反。
- Secret 是否复制完整。
- WebDAV 凭证是否已撤销。
- WebDAV 凭证是否绑定了目标目录。

### 6.2 WebDAV 返回 403

检查：

- 当前操作是否超出绑定目录。
- 上传是否缺少新建权限。
- 覆盖、重命名是否缺少更新权限。
- 删除是否缺少删除权限。

### 6.3 S3 返回 AccessDenied

检查：

- S3 Access Key ID 和 Secret 是否正确。
- Region 是否与服务端配置一致。
- 客户端是否使用 path-style。
- bucket 是否为 `personal`、`apps` 或 `services`。
- 对象 key 是否落在凭证授权的目录前缀下。
- 操作是否具备对应权限。

### 6.4 S3 客户端看不到 bucket

Warehouse 会按凭证授权范围过滤 bucket。比如凭证授权 `/personal/backup`，`ListBuckets` 只会看到 `personal`；凭证授权 `/apps/demo`，只会看到 `apps`；凭证授权 `/services/reporting`，只会看到 `services`。

### 6.5 上传失败或中途断开

检查：

- 当前用户额度是否足够。
- 反向代理是否限制请求体大小或上传超时。
- S3 Endpoint 是否指向 active 节点。
- 客户端是否因为断点续传、覆盖或临时文件操作需要额外权限。

### 6.6 删除后额度没有下降

如果是 Web 页面或 WebDAV 删除，文件会进入回收站，仍然占用额度。需要在回收站永久删除或清空回收站后，额度才会下降。

如果是 S3 删除，删除是永久删除，正常情况下会释放对应存储占用。

## 7. 安全建议

- 不要把账号密码配置到自动化脚本中，改用 WebDAV 或 S3 凭证。
- 每个客户端、脚本或集成任务使用独立凭证。
- 每个凭证只授权一个明确目录前缀。
- Secret 只保存到密码管理器、CI Secret 或系统凭证存储中。
- 定期轮换：新建新凭证，客户端切换成功后撤销旧凭证。
- 已撤销且不再需要展示的凭证可以删除。
- 不要在日志、截图、Issue、PR 或聊天记录中粘贴 Secret。
